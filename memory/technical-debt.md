# technical-debt.md — What we deferred, and why

> The honest list of shortcuts, their cost, and when we'll pay them off. A debt
> entry is a promise to come back — log it, don't hide it.

> No implementation yet → no accrued debt. This template is here so the first
> shortcut has a home.

## Template

```
## [Short title] — <date incurred> — <phase> — <severity: low|med|high>
- **What:** the shortcut taken.
- **Why:** the constraint that justified it (time, blocked-on, good-enough).
- **Cost / risk:** what breaks or slows if left unpaid.
- **Payoff plan:** when/how we revisit (which phase/trigger).
```
